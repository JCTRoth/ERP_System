== Overview

Scripts are organised into subfolders by purpose.
Always run scripts from the **repository root**.

[cols="1,3"]
|===
| Subfolder | Purpose

| link:dev/[dev/]
| Start / stop the local Docker Compose stack

| link:build/[build/]
| Build all services and container images

| link:k3s/[k3s/]
| Deploy and manage the stack on a local K3s cluster

| link:test/[test/]
| Integration, federation, and E2E tests

| link:translation/[translation/]
| Sync and generate i18n / translation seed data

| link:setup/[setup/]
| Developer machine setup (Ubuntu/Debian)

| link:deployment/[deployment/]
| Push images to GHCR and deploy to a production server

| link:document_test_utils/[document_test_utils/]
| Utilities for generating and syncing test documents/templates
|===

Each subfolder contains its own `README.md` with full usage instructions.

== Quick-start (local development)

[source,bash]
----
# Start the complete stack
./scripts/dev/start-local.sh

# Run all integration tests
./scripts/test/test-all-services.sh

# Stop the stack
./scripts/dev/stop-local.sh
----

== Quick-start (local Kubernetes / K3s)

[source,bash]
----
# Full first-time deployment
./scripts/k3s/k3s-local-deploy.sh full

# Quick rebuild after PC restart
./scripts/k3s/k3s-quick-restart.sh

# Tear everything down
./scripts/k3s/k3s-local-stop.sh clean
----

== Build & push to GHCR

[source,bash]
----
# Build all images locally
./scripts/build/build-all-containers.sh

# Push to GitHub Container Registry
./scripts/deployment/deploy-to-registry.sh --username JCTRoth --token ghp_xxx --version 1.0.0
----

== Access points (after local start)

* *Frontend*: http://localhost:5173
* *Apollo Gateway*: http://localhost:4000/graphql
* *Default login*: admin@erp-system.local / Admin123!
